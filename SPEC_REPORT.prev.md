# Отчёт по итерации

**Версия:** 0.32.0
**Дата:** 2026-04-03

## Выполнено

### Must Have 1: UI — Кнопка «Собрать чаты» + модальное окно
Реализовано. Кнопка рядом с «Загрузить чаты». Модалка содержит поле названия группы, textarea для запроса, чекбоксы 12 площадок с иконками ценового тира (💚/🟡/🔴). Площадки без API-ключа задизейблены с пометкой. Кнопка «Начать поиск» запускает процесс.

### Must Have 2: Новый статус группы `scraping`
Реализовано. Добавлен в GroupStatus enum. Группа в статусе scraping отображается с индикатором прогресса — видно какие площадки обработаны, какие в процессе. После завершения скрапинга + дедупликации статус переходит в `pending`.

### Must Have 3: Спецификации площадок (Platform Specs)
Реализовано. 12 площадок со структурированными спецификациями: метод доступа (API/HTTP/Playwright), URL-паттерны, формат ответа, rate limits, необходимость API-ключа. Спецификации хранятся в коде.

### Must Have 4: Движок поиска (Search Engine)
Реализовано. AI генерирует поисковые запросы из текста пользователя. Для каждой площадки выполняется поиск по спецификации. Собираются chat_ref со всех площадок, единая дедупликация, создание группы с уникальными результатами.

### Must Have 5: Админка — настройки площадок
Реализовано. Для каждой площадки: API-ключ (masked), стоимость за запрос в USD, toggle вкл/выкл. Глобальный мультипликатор стоимости. Новая секция «Площадки поиска» в админке.

### Must Have 6: Биллинг поиска
Реализовано. Стоимость = (токены AI + API-запросы × цена) × мультипликатор. Проверка баланса перед началом. Списание по факту. Транзакция с type="search" в ai_transactions.

## Не выполнено

Все Must Have реализованы. Nice to Have (предварительная оценка стоимости, остановка поиска, подсказки запросов) — не реализованы в этой итерации.

## Итог

Реализована полная функциональность AI-driven поиска Telegram-чатов по 12 внешним площадкам. Все 6 Must Have требований из SPEC.md выполнены и протестированы: UI с модальным окном, статус scraping с прогрессом, спецификации площадок, движок поиска с AI-генерацией запросов, админка с настройками площадок и мультипликатором, биллинг с проверкой баланса. Функциональное тестирование — 6/6 PASS. Backend — 2326 тестов проходят. CI — все 5 jobs успешны. Исправлено 6 багов, найденных при smoke-тестировании.
