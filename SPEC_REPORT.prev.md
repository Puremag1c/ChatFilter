# Отчёт по итерации

**Версия:** 0.27.0
**Дата:** 2026-03-22

## Выполнено

### Must Have 1: CSRF ломает логин
Исправлено. Проблема была в том, что CSRF-токен из сессии не передавался при POST /login. Исправлена логика middleware — токен корректно проверяется через form-field. Добавлен регрессионный тест. Дополнительно убран /login из CSRF exempt paths для устранения login CSRF уязвимости.

### Must Have 2: Светлая тема на странице логина
Исправлено. CSS-стили светлой темы корректно применяются на login.html.

### Must Have 3: CLI-команда reset-password
Реализовано. Команда `chatfilter reset-password <user> <password>` позволяет сбросить пароль из терминала. Реализованы: минимальная длина пароля, скрытие пароля из списка процессов, обработка блокировки SQLite при запущенном приложении, тесты.

## Не выполнено

Нет — все Must Have требования реализованы. Nice to Have отсутствовали.

## Итог

Устранён критический баг с CSRF, блокировавший логин. Исправлена светлая тема на странице логина. Добавлена CLI-команда сброса пароля с защитой от утечки пароля и обработкой edge cases. Все тесты проходят, визуальное и функциональное тестирование завершено успешно.
